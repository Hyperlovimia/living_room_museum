# 6月8日工作日志

## 项目目标

项目定位为 PC 端第一人称沉浸式中华起居文化数字展厅，重点放在空间漫游、展品信息展示、音视频讲解、传送和找一找任务等交互功能上。

## 今日完成内容

1. 接入并学习使用 Unity-Skills，通过本地 Unity Editor REST 服务读取场景、移动物体、保存场景和检查编译状态。
2. 梳理当前场景结构，确认主要场景为 `Assets/Scenes/LivingRoom.unity`，核心对象包括 `出生点`、`书房`、`PlayerController`、`大厅` 等。
3. 优化起始房间墙面文字：
   - 修复文字在 Game View 中左右镜像的问题。
   - 将字体缩小，减少遮挡和视觉压迫。
   - 将一次性显示改为类似打字机的逐字出现效果。
4. 优化起始房间按钮视觉：
   - 隐藏原本突兀的墙面凸起按钮渲染，仅保留交互碰撞区域。
   - 增加世界空间按钮说明牌，区分“进入书房”和“找一找”。
5. 调整传送点：
   - 将进入书房后的传送目标移动到书房中心。
   - 将返回大厅目标移动到起始房间中心。
   - 修复返回按钮被 `PlayerMovementBounds` 夹回书房边缘的问题。
6. 初步实现展品信息系统扩展：
   - 扩展 `ExhibitInfo`，支持图文、音频和视频式讲解字段。
   - 扩展 `ExhibitInteractionController`，支持无真实视频素材时的脚本化视频讲解占位。
   - 新增 `MuseumNextStepContentBootstrap`，用于运行时生成展品交互热点和返回按钮。
   - 新增 `TeleportClickInteractable`，用于书房内返回起始房间。
   - 新增 `WelcomeSequenceController`，用于进入游戏后的欢迎视频式界面。

   - 现有音频资源仍是占位性质，不能作为最终讲解音频。
   - 需要为至少 4 个展品挂载真实讲解音频，并逐个测试点击后是否播放。
2. 展品信息热点还没有准确挂载到书房内原本存在的真实物品上：
   - 当前做法是按大致空间位置生成透明点击热区。
   - 需要重新定位书房内真实家具/器物模型，将交互范围贴合到正确物品位置。
3. 找一找任务尚未完成开发：

   - 还需要完成任务开始、目标提示、寻找状态栏、正确物品判定、成功提示和返回起始房间的完整流程。
4. 书房返回大厅按钮视觉已有初步优化，但位置和朝向仍需要结合 Game View 实测微调。
